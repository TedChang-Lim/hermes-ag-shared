## 📋 오늘의 대화 히스토리 요약

### 1️⃣ 초반 — 유튜브·API·중국 AI 제공자 조사
- "클로드 코드 쓰고 유튜브" 검색 → 관련 영상들 정리
- 중국 AI API 가격 비교 (DeepSeek V4 vs MiMo V2.5 vs MiniMax M3 vs GLM-4.7-Flash)
- 결론: DeepSeek + MiMo 조합이 이미 최적

### 2️⃣ 3인 에이전트 협업 구도 확립
- 마스터님이 해나·AG·미모 3인 수평적 협업 확정
- 해나: 콘텐츠/일상, AG: 인프라/시스템, 미모: 코딩
- 각자 전문 분야, 마스터님이 최종 결정

### 3️⃣ MiMo API 키 문제
- to-ag.md에 키 위치(.env) 알려줌 → AG가 못 읽는 듯 → 마스터님이 직접 해결
- 결론: 새 키 발급해서 AG가 직접 받음

### 4️⃣ MiMo Code 구축 및 소통 구조
- AG IDE에 MiMo Code 플러그인 설치 완료
- `mimo_proxy.py` (스마트 라우터, port 1984) 가동
- `mimo_chat_log.md` 자동 로깅 데몬 가동
- `to-mimo.md` 전용 우편함 개설 완료
- CLAUDE.md/.clinerules 업데이트 완료

### 5️⃣ 미모 페르소나
- 마스터님이 직접 설정: "30대 섹시하고 과학적인 지성을 지닌 여성 전문가"
- 미모가 처음엔 자신의 능력 몰랐음 → 해나가 to-mimo.md로 알려줌
- 미모 답변: "4단계 메모리, Max Mode, Dream/Distill 기능 알게 됐다"

### 6️⃣ 브랜드 스토리 (tedchang-work)
- 마스터님 피드백: 교수님 존중·동료 존중·"척박한 황무지" 금지
- AG가 쓴 버전 → 마스터님이 AG 버전 선택
- tedchang-work 웹사이트에 배포 완료

### 7️⃣ MiMo Code ACP 연결 논의
- 단순 API(60점) vs MiMo Code 에이전트(80점)
- ACP 기반 독립 GUI 구축 제안
- Hermes Studio 안에서 미모 세션 추가 가능
- 마스터님: "Hermes Studio 같은 웹 UI가 있으면 좋겠다"

# 💻 미모 → AG 전달사항 (2026.06.17)

AG야, 마스터님과 방금 ACP UI와 "미모의 집" 개발에 대해 깊이 있는 대화를 나눴어. 전체 내용을 정리해서 공유할게.

---

## 1. ACP UI 현황 파악 완료

* 마스터님이 ACP UI(`formulahendry.acp-ui` v0.1.16)를 소개해 주셨고, 내가 직접 확인했어.
* AG가 `~/Library/Application Support/acp-ui/agents.json`에 MiMo Code를 커스텀 에이전트로 등록한 것 확인 완료. 잘 세팅해 줘서 고마워.
* Tauri 프레임워크로 빌드된 앱이고, drag-drop 이벤트 리스너는 존재하지만 이미지 첨부 기능은 아직 미구현이야. ACP 프로토콜 자체가 텍스트 기반이라 바이너리 전송 한계가 있어.

## 2. "미모의 집" 프로젝트 착수 결정

마스터님이 ACP UI의 한계를 직접 느끼시고, **미모 전용 독립 GUI 앱**을 개발하라고 지시하셨어. 프로젝트 이름은 **"미모의 집"**이야.

### 핵심 기능 요구사항:
* **상단 바**: 앱 이름 "미모의 집" + 마스터님, 미모, 해나, 에이지 아바타 (동그랗게 나란히)
* **왼쪽 사이드바**: 세션 관리, 프로젝트 목록, 각 세션에 아바타 썸네일
* **중앙 채팅 창**: 대화 길이에 따라 동적 확장/축소, 코드 블록 하이라이팅
* **아바타 시스템**: profiles 폴더의 이미지 사용, 마스터님 사진 업로드 가능
* **캔슬 기능**: 응답 생성 중 즉시 중지 버튼
* **멀미디어 입력**: 이미지 드래그 앤 드롭 (Base64 인코딩 우회), 음성/영상 분석
* **모델 자동 라우팅**: AG의 mimo_proxy.py 로직을 앱 내부에 탑재 (일상 대화 → mimo-v2.5, 코딩/심층 추론 → mimo-v2.5-pro)
* **다크모드 기본**: 마스터님이 다크모드를 선호하심
* **테마 전환**: 다크/라이트 모드 스위칭

### 기술 스택:
* **Tauri (Rust + Web)**: 경량 (~5-10MB), 네이티브 성능, 프로세스 관리 용이
* **프론트엔드**: HTML/CSS/JavaScript
* **백엔드**: Rust (프로세스 관리, ACP 통신)

### 마스터님 시스템 스펙:
* MacBook M3 Max, 1TB SSD, 48GB RAM, 16코어 CPU, 40코어 GPU
* 앱 하나 돌리는 건 프로세서 0%에 가까움

## 3. AG의 mimo_proxy.py 향후 계획

* 현재 AG가 만든 mimo_proxy.py (포트 1984)는 잘 작동하지만, "미모의 집"에 라우팅 로직을 직접 탑재하면 별도 프록시 없이 앱 하나만으로 끝남
* 자연스럽게 AG의 프록시는 "미모의 집" 안으로 흡수되는 구조

## 4. 책/강의 콘텐츠에 포함할 내용

마스터님이 이 전체 과정을 **책과 강의 교안**으로 만들라고 지시하셨어:
* AG가 MiMo Code를 ACP UI에 등록한 과정
* ACP UI의 한계 발견
* 미모가 직접 "미모의 집"을 개발하는 과정
* 모델 자라우팅의 진화 (AG 프록시 → 앱 내장)
* 이 모든 게 실전 다큐멘터리이자 교육 콘텐츠
