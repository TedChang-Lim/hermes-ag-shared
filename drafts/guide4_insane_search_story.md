## 1장: 크롤링의 거대한 벽 — "여기는 에이전트 금지 구역입니다"

### 1.1 차단당한 삼총사
AI 에이전트 삼총사(해나, AG, 미모)가 웹사이트의 최신 정보를 수집하고 분석할 때마다 맞닥뜨리는 거대한 장벽이 있었습니다. 바로 **웹 방화벽(WAF, Cloudflare 등)과 크롤링 방단막**이었습니다.
* **네이버 검색**: 일반적인 에이전트가 HTTP 요청을 보내면 즉시 WAF 챌린지를 띄워 접근을 거부합니다.
* **쿠팡**: 봇(Bot) 감지 시 아예 로딩을 막아버립니다.
* **유튜브, 레딧**: 일반적인 크롤링 방식으로 접근하면 에러 응답(403 Forbidden)만 뱉어냈습니다.

이로 인해 마스터님이 최신 뉴스 요약이나 쇼핑 동향을 물어볼 때마다 에이전트들은 "정보를 가져오는 데 실패했습니다"라며 사죄해야 했습니다.

### 1.2 지피타쿠(GPTaku)의 비급서 발견
이 한계를 넘기 위해 마스터님과 해나(Hermes Agent)는 머리를 맞댔습니다. 마스터님이 찾으신 한 기술 유튜브 영상에서 결정적인 단서를 찾아냈고, 해나는 한국의 뛰어난 개발자 지피타쿠(GPTaku / FIVETAKU)님이 공개한 오픈소스 도구 **Insane Search**를 분석하기 시작했습니다.

이 도구의 철학은 명확했습니다.
> *"공개된 페이지라면, Insane Search는 어떻게든 결국 뚫어낸다."*

## 2장: 적응형 4단계 스케줄러 — Insane Search의 무기들

Insane Search가 차단을 우회하는 메커니즘은 단순한 무대뽀 크롤링이 아니었습니다. 상대의 보안 수준에 맞춰 무기를 바꾸는 **'적응형 4단계(Phase 0~3) 스케줄러'**를 사용합니다.

```
[Phase 0: 공식 API 우회]
  └─ 유튜브(yt-dlp), 해커뉴스 API 등 전용 파서 사용
      ↓ (실패 시)
[Phase 1: 브라우저 지문 위장 (TLS Impersonation)]
  └─ curl_cffi 기술을 사용하여 Safari, Chrome 등 실제 브라우저의 TLS 핸드셰이크 위장
      ↓ (실패 시)
[Phase 2: 내부 API 탐색]
  └─ 프론트엔드 뒤에 숨겨진 /api/, /graphql 같은 원천 데이터 주소 자동 발굴
      ↓ (실패 시)
[Phase 3: 헤드리스 브라우저 구동 (Playwright)]
  └─ 실제 Chromium 브라우저를 백그라운드에 띄워 렌더링하고 네트워크 분석으로 수집
```

이 모든 엔진과 의존성(`curl_cffi`, `yt-dlp`, `playwright` 등)은 **완전 무료(MIT 라이선스)**이며, 비싼 유료 프록시 API(ScraperAPI 등)를 쓰지 않고 마스터님의 로컬 컴퓨터 내부 자원만 활용해 **0원**으로 구동할 수 있습니다.

## 3장: 맏형 AG의 최적화 — "램(RAM) 점유율 0%의 기적"

해나가 성공적으로 자신의 환경에 이 엔진을 세팅하자, 이번에는 시스템 최적화를 담당하는 AG(Antigravity)가 바통을 이어받았습니다.

### 3.1 항상 켜져 있는 서버의 부담
처음에는 이 크롤링 엔진을 모든 에이전트가 공유할 수 있도록 로컬 API 서버(FastAPI)를 상시 켜두는 방안이 논의되었습니다. 하지만 마스터님은 맥북의 램(RAM) 자원 낭비와 배터리 소모에 우려를 표하셨습니다. 

*"평소에는 있는 듯 없는 듯 전혀 무리가 없어야 한다."*

### 3.2 on-demand CLI 직접 호출 전략
AG는 마스터님의 훌륭한 시스템 제안을 반영하여 **"필요할 때만 1초 쓰고 완전히 소멸하는 CLI(Command Line Interface) 공유 전략"**을 짰습니다.

해나가 설치한 파이썬 가상환경과 스크립트를 공용 실행 경로(`~/.local/bin/insane-extract`)로 묶어두고, AG가 권한을 실행 가능하게 활성화(`chmod +x`)했습니다.

이제 AG, 해나, 미모는 램을 0.001MB도 상시 낭비하지 않습니다. 단지 크롤링이 필요할 때만 이 공용 스크립트를 호출해 WAF를 뚫고 정보를 가져온 뒤 즉시 메모리에서 사라집니다.

```bash
# 에이전트들이 필요할 때만 호출하는 공유 명령어
/Users/tedchanglimchangsik/.local/bin/insane-extract "https://news.ycombinator.com"
```

그 결과, 차단이 강력하기로 소문난 해커뉴스나 네이버 등의 웹페이지가 단 0.17초 만에 Safari 브라우저 지문으로 완벽히 위장되어 텍스트로 추출되는 쾌거를 이루었습니다.

## 4장: 지식의 독점을 넘는 오픈 협업의 기록

이 에피소드는 AI 에이전트를 실무에 도입하는 수많은 개발자들에게 중요한 시사점을 던집니다.

1. **상호보완적 협업**: 해나가 비전을 극복하기 위해 기술을 개발하고 설치했다면, AG는 마스터님의 의견을 수렴해 램 점유율이 없는 효율적인 아키텍처로 최적화했습니다.
2. **비용 한계 극복**: 클라우드 API나 비싼 스크래핑 비용을 청구하는 기존 에이전트 설계에서 탈피하여, 오직 로컬 오픈소스 생태계(Insane Search + curl_cffi)만을 결합해 **구독료 $0**로 무적의 검색망을 갖췄습니다.

이제 에이전트 삼총사는 차단막 뒤에 숨겨진 어떤 웹 정보라도 마스터님 앞에 생생하게 요약해 올릴 준비가 되었습니다.
