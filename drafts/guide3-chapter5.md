## 5.1 양자화(Quantization)란?

AI 모델은 수십억 개의 숫자(파라미터)로 이루어져 있습니다. 이 숫자들은 보통 16비트(FP16) 또는 32비트(FP32) 정밀도로 저장됩니다.

**양자화는 이 숫자들의 정밀도를 낮춰서 파일 크기를 줄이는 기술입니다.**

| 정밀도 | 비트 수 | 파일 크기 (35B 모델 기준) | 속도 |
|:------|:------:|:-----------------------:|:----:|
| FP16 (원본) | 16비트 | 약 70GB | ❌ 맥북에서 실행 불가 |
| Q8_0 | 8비트 | 약 35GB | 느림 |
| Q4_K_M | 4비트 | 약 18GB | 보통 |
| **Q3_K** | **3비트** | **약 14GB** | **빠름** |
| Q2_K | 2비트 | 약 10GB | 가장 빠름 (but 품질 저하) |

> **비유:**
> 원본 FP16 = 4K 화질의 영화 (70GB)
> Q4 양자화 = FHD 화질 (18GB) — 육안으로 차이 거의 못 느낌
> Q2 양자화 = 360p 화질 (10GB) — 볼 순 있는데... 음...

## 5.2 GGUF 양자화 포맷 완벽 정리

GGUF는 llama.cpp 생태계에서 사용하는 표준 양자화 포맷입니다. 파일 이름에 포함된 코드가 무엇을 의미하는지 알아두면 모델 선택이 훨씬 쉬워집니다.

### 파일명 해석법

```
Qwen-3.5-3B-Q4_K_M.gguf
└──────┘ └─┘ └──┴─┘
   모델명  크기  양자화
```

### 양자화 등급별 특징

| 코드 | 비트 | 파일 크기 비율 | 품질 | 추천 용도 |
|:---:|:---:|:------------:|:---:|:---------|
| **Q8_0** | 8비트 | 100% (기준) | ⭐⭐⭐⭐⭐ 최상 | 충분한 RAM 있을 때 |
| **Q6_K** | 6비트 | 75~80% | ⭐⭐⭐⭐ 우수 | 고품질 + 적당한 크기 |
| **Q5_K_M** | 5비트 | 65~70% | ⭐⭐⭐⭐ 좋음 | ⭐ M3 Max 48GB 추천 |
| **Q5_0** | 5비트 | 65% | ⭐⭐⭐⭐ | K variant보다 약간 낮음 |
| **Q4_K_M** | 4비트 | 50~55% | ⭐⭐⭐ 좋음 | **가장 보편적, 강력 추천** |
| **Q4_K_S** | 4비트 | 45~50% | ⭐⭐⭐ | K_M보다 약간 낮음 |
| **Q4_0** | 4비트 | 45~50% | ⭐⭐⭐ | 기본 4비트 |
| **Q3_K_L** | 3비트 | 40~45% | ⭐⭐ | 큰 모델용 (35B+) |
| **Q3_K_M** | 3비트 | 35~40% | ⭐⭐ | ⭐ M3 Max 48GB 추천 (큰 모델) |
| **Q3_K_S** | 3비트 | 30~35% | ⭐⭐ | 가장 작은 3비트 |
| **Q2_K** | 2비트 | 25~30% | ⭐ | 비상용, 품질 저하 있음 |

### K 시리즈 이해하기

`Q4_K_M`에서 `K`는 **K-quant**를 의미합니다. K-quant는 모델의 각 부분(레이어)에 따라 다른 정밀도를 적용하는 **스마트 양자화** 방식입니다.

| 접미사 | 의미 | 설명 |
|:-----:|:----|:-----|
| **K_M** | K-quant Medium | **표준 K-quant** — 대부분의 경우 최적 |
| **K_S** | K-quant Small | K_M보다 약간 낮은 품질, 더 작은 크기 |
| **K_L** | K-quant Large | 더 높은 품질, 조금 더 큰 크기 |

> **결론: `Q4_K_M` 또는 `Q5_K_M`을 선택하면 됩니다.**
> 특별한 이유가 없다면 이 두 가지 중에서 고르세요.

## 5.4 다운로드 팁

### 어디서 다운로드하나?

```bash
# HuggingFace에서 검색
# https://huggingface.co/models?search=gguf

# 추천 저장소
# - Qwen: https://huggingface.co/Qwen
# - gemma: https://huggingface.co/google/gemma-4
# - OpenYourMind (Uncensored): https://huggingface.co/OpenYourMind
```

### CLI로 다운로드

```bash
# pip 방식 (범용)
pip install -U "huggingface_hub[cli]"

# 또는 홈브루 방식
brew install hf

# 다운로드 예시
huggingface-cli download Qwen/Qwen-3.5-3B-GGUF \
  qwen-3.5-3b-q4_k_m.gguf \
  --local-dir ~/Downloads/
```

### 다운로드 속도

| 파일 크기 | 500Mbps 기준 | 100Mbps 기준 |
|:---------|:-----------:|:-----------:|
| 2GB (3B 모델) | 30초 | 2~3분 |
| 5GB (7B 모델) | 1~2분 | 5~7분 |
| 17GB (35B MoE) | **5~6분** | 20~25분 |

> 이 가이드의 저자는 500Mbps 환경에서 17GB 모델 다운로드에 약 5분 30초가 소요되었습니다.

## 5.5 실전: 모델 비교 테스트

### 같은 모델, 다른 양자화 비교

실제로 Qwen 3.5 7B를 다른 양자화로 실행했을 때의 차이:

| 양자화 | 파일 크기 | 속도 (M3 Max) | 품질 느낌 |
|:-----:|:--------:|:------------:|:---------|
| Q8_0 | 7.5GB | 48 tok/s | 원본과 거의 동일 |
| Q6_K | 5.7GB | 52 tok/s | 원본과 거의 동일 |
| **Q5_K_M** | **4.9GB** | **55 tok/s** | **차이 못 느낌** |
| Q4_K_M | 4.2GB | 58 tok/s | 약간의 차이 (번역 품질 등) |
| Q3_K_M | 3.3GB | 62 tok/s | 차이 느껴짐 |
| Q2_K | 2.5GB | 68 tok/s | 품질 저하 체감 |

> **Q5_K_M과 Q4_K_M 사이에서 고민된다면:**
> - 일반 채팅/코딩: Q4_K_M (충분함, 더 빠름)
> - 번역/창작/정밀 작업: Q5_K_M (약간 더 나은 품질)
