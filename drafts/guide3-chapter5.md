## 5.1 양자화(Quantization)의 수학적 압축 메커니즘

언어 모델의 지식은 수십억 개의 가중치 파라미터(실수형 데이터)로 기술되어 있습니다. 이 가중치들은 원본 빌드 상태에서 일반적으로 16비트(FP16) 또는 32비트(FP32) 부동 소수점 규격을 따릅니다.

**양자화는 이 정밀도 비트수를 줄여 연산 요구랑과 파일 크기를 극적으로 낮추는 인코딩 변환 기술입니다.**

| 변환 포맷 | 정밀도 비트 | 35B 모델 기준 크기 | 하드웨어 요구 사항 및 추론 평가 |
|:------|:------:|:-----------------------:|:----:|
| **FP16 (원본)** | 16비트 | 약 70GB | ❌ 로컬 맥북 통합 RAM 한계 초과 (추론 불가) |
| **Q8_0** | 8비트 | 약 35GB | 메모리 점유 높음, 고사양 사양 요구 |
| **Q4_K_M** | 4비트 | 약 18GB | 성능 저하 대비 효율 최적화 (표준) |
| **Q3_K** | 3비트 | 약 14GB | 대용량 모델 구동을 위한 최적의 타협점 |
| **Q2_K** | 2비트 | 약 10GB | 초고속 추론 지원, 단 정보 손실로 인한 환각 현상 증가 |

로컬 장비에서 양자화 등급을 영리하게 선택해야 하는 근본적인 이유는 **'메모리 누출 차단'**과 **'가상 메모리 스왑 방지'**에 있습니다. 만약 가용 RAM 크기보다 큰 모델을 강제로 구동하면 운영체제는 디스크 영역을 가상 메모리로 빌려 쓰는 스왑(Swap) 처리를 시작합니다. 이는 추론 속도를 1/10 이하로 떨어뜨릴 뿐 아니라, 디스크 스왑 파일에 암호화되지 않은 기밀 잔재가 임시 기록되는 보안 약점을 낳습니다. 가용 통합 물리 램 영역 내부에 완벽히 격리 안착될 수 있는 핏(Fit)한 모델 선정이 안전한 로컬 AI의 대전제입니다.

## 5.2 GGUF 양자화 클래스 명세

GGUF 명칭 규칙은 가중치의 압축 방식과 타겟 정밀도를 투명하게 밝힙니다.

### 파일 명세 포맷

```
Qwen-3.5-3B-Q4_K_M.gguf
├──────┘ ├─┘ └──┴─┘
 모델명  크기  양자화 규격 (4비트 K-quant Medium)
```

### 규격별 연산 특성

| 압축 코드 | 실효 비트수 | 원본 대비 용량 비율 | 추론 정밀도 보존력 | 하드웨어 타겟 매핑 |
|:---:|:---:|:------------:|:---:|:---------|
| **Q8_0** | 8비트 | 100% (기준) | ⭐⭐⭐⭐⭐ 최상 | 통합 메모리가 여유로울 때 채택 |
| **Q6_K** | 6비트 | 75~80% | ⭐⭐⭐⭐ 매우 우수 | 퀄리티 지향 창작 및 정밀 분석용 |
| **Q5_K_M** | 5비트 | 65~70% | ⭐⭐⭐⭐ 우수 | 성능 대 용량 절충, M3 Max 48GB 탑재 권장 |
| **Q4_K_M** | 4비트 | 50~55% | ⭐⭐⭐ 준수 | **가장 범용적이며 대중적인 추천 프로필** |
| **Q4_0** | 4비트 | 45~50% | ⭐⭐⭐ 기본형 | 가상 런타임 검증용 |
| **Q3_K_M** | 3비트 | 35~40% | ⭐⭐ 타협 가능 | 30B 이상 고체급 모델의 저사양 구동용 |
| **Q2_K** | 2비트 | 25~30% | ⭐ 한계 노출 | 특수 환경 성능 테스트 및 시뮬레이션용 |

### 스마트 양자화: K-Quant 아키텍처
`Q4_K_M` 등에 명시된 **K**는 핵심적인 가중치 레이어(어텐션 블록 등)는 고정밀로 보존하고, 영향도가 떨어지는 가중치 블록은 강하게 압축하는 가변 정밀도 압축 기술을 뜻합니다.
- **K_M (Medium)**: 하드웨어 최적화 균형 지점입니다. 특별한 사유가 없는 한 이 등급을 권장합니다.
- **K_S (Small)**: 메모리 여유가 극도로 부족할 때 용량을 소폭 축소하기 위해 선택합니다.
- **K_L (Large)**: 용량이 커지더라도 언어 해석 품질을 끝까지 지키고자 할 때 씁니다.

## 5.3 RAM 하드웨어 매핑 테이블

물리 통합 RAM의 상한선 안에서 구동해야 데이터 프라이버시(Mythos 샌드박스)가 보존됩니다.

| 물리 RAM 스펙 | 안정 구동 한계 크기 | 최적 권장 압축 규격 | 예시 모델 |
|:-------:|:------------:|:-----------:|:---------|
| **8GB** | 4B 모델 내외 | Q4_K_M | Qwen 3.5 3B, Gemma 4 4B |
| **16GB** | 7B ~ 9B 모델 | Q4_K_M / Q5_K_M | Qwen 3.5 7B, Gemma 4 8B |
| **36GB** | 14B ~ 20B 모델 | Q4_K_M | Qwen 3.5 14B, CodeQwen 14B |
| **48GB ⭐** | 30B ~ 35B 모델 | **Q3_K_M 또는 APEX I-Compact** | **Qwen3.6-35B-A3B (MoE)** |
| **64GB** | 40B ~ 70B 모델 | Q3_K_M | Llama 3 70B Q3 |
| **128GB** | 70B ~ 120B 모델 | Q4_K_M | Llama 3 70B Q4, Mixtral MoE |

## 5.4 다운로드 및 보존 자동화

### Hugging Face CLI 다운로드 기법
터미널 환경에서 끊김 없이 고속으로 안정 수신을 받아내는 전개 방식입니다.

```bash
# 1. Hugging Face CLI 도구 업데이트
pip install -U "huggingface_hub[cli]"

# 2. 로컬 디렉토리 타겟 지정 후 다운로드 개시
huggingface-cli download Qwen/Qwen-3.5-7B-Instruct-GGUF \
  qwen-3.5-7b-instruct-q5_k_m.gguf \
  --local-dir ~/Library/Application\ Support/Jan/data/llamacpp/models/Qwen-3.5-7B-Local
```

## 5.5 정밀 비교 벤치마크 (Qwen 3.5 7B 실측)

M3 Max 48GB 조건 하에서 양자화 등급별 출력 속도와 정밀도 왜곡 정도의 관계식입니다.

| 양자화 등급 | 실질 스토리지 용량 | 가속 속도 (M3 Max) | 한글 번역/코딩 추론 정밀도 보존력 |
|:-----:|:--------:|:------------:|:---------|
| Q8_0 | 7.5GB | 48 tok/s | 원본 FP16 모델과 구별 불가능 |
| Q6_K | 5.7GB | 52 tok/s | 원본 보존율 99% 이상 |
| **Q5_K_M** | **4.9GB** | **55 tok/s** | **실제 인지 불가 수준의 준수함** |
| Q4_K_M | 4.2GB | 58 tok/s | 고난도 추론 시 일부 특수 기호 마크다운 유실 가능성 존재 |
| Q3_K_M | 3.3GB | 62 tok/s | 맥락 어휘 어색함 발생하기 시작함 |
| Q2_K | 2.5GB | 68 tok/s | 번역 어투의 부자연스러움 및 잦은 무한 루프 발생 |

### 최적의 결정 전략
안정적인 언어 작문 및 정밀 코딩 빌드가 요구되는 프로덕션 목적에는 **Q5_K_M** 규격을 선택하며, 극강의 속도와 경량 메모리 점유가 중심인 범용 작업에는 **Q4_K_M**이 가장 훌륭한 선택지입니다.
