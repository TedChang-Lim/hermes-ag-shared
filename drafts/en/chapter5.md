## 5.1 Background on Adopting the "Value-for-Money Trio"

The authoring team behind this guide is a 30-year veteran commercial photographer and planner who runs two or more AI agents (Hena via Hermes Agent, AG via AntiGravity, and MiMo) 24 hours a day to conduct business.

In the past, high-performance large models (GPT-4o, Claude Pro, etc.) alone incurred roughly ₩300,000 per month in fixed subscription fees. As the technology matured, however, the per-unit cost of coding and data-collection APIs skyrocketed exponentially, imposing a severe constraint on solo creators.

That is when we executed a wholesale architectural pivot toward the **"Cheapest Brains"** strategy — an approach that drives costs toward near-zero while preserving guaranteed performance.

## 5.3 Three Pillars That Maximized Cost Efficiency

### ① Cheapest Brains Architecture
Instead of relying on a single expensive model, we coordinated a collaborative trio of ultra-lightweight Flash and Base models — each priced at $0.14 or less per million input tokens — dismantling the high-cost structure entirely.

### ② Open Design Framework
By injecting design guidelines from Stripe and Vercel into lightweight, text-only models that have no vision capability, we produced sophisticated dark-mode + gold-theme proposals without incurring any additional multimodal costs.

### ③ Insane Search CLI Crawling Wrapper
Rather than running a heavy, always-on backend server that devours memory around the clock, we used an on-demand CLI architecture that spins up only when information is needed and vanishes without a trace afterward. This achieved **0% standby RAM occupancy and $0 in additional proxy subscription fees** while breaching formidable Web Application Firewalls (WAFs).

## 5.5 Frequently Asked Questions and Solutions

### Q: Isn't 55 million tokens per day an excessive figure for an ordinary user?
Every time an agent reads local source code to understand cross-file structural relationships, or crawls and summarizes the latest news, context windows spanning millions of tokens flow through in an instant. While 55 million tokens equates to roughly 55 full-length books, lightweight models and automatic caching allow us to process this staggering volume effortlessly — as naturally as breathing — with zero cost burden.

### Q: Doesn't performance fall short of large models, resulting in subpar output?
Judged in isolation by coding-benchmark scores, the trio may not match the top-ranked model. However, when agents collectively adhere to the shared contract of the **Open Design framework** and instantly source WAF-blocked information via the **Insane Search CLI** for cross-verification, this team-play architecture can build premium products that surpass the output of enterprise solutions costing tens of millions of won.

## 5.6 Overall Assessment and Final Recommended Configurations

### Strengths
- **Extreme cost control**: Overwhelming value sustained at under $5 per month
- **Speed and agility**: Flash models' characteristic low-latency, rapid real-time feedback
- **RAM friendliness**: Zero burden on local hardware thanks to the on-demand CLI approach
- **Verified UI**: Guaranteed high visual quality through design-framework injection

### Weaknesses and Limitations
- **Initial setup time**: Requires basic engineering skills — API key integration, local CLI path registration, etc.
- **Training-data ceiling**: Excessively long and deep historical contexts exceeding 128K tokens occasionally require support from a reasoning model (Pro)
